Nesse capítulo trataremos de formulários. A tag contêiner para formulários é < FORM> < /FORM>. Dentro do < FORM> colocamos ACTION e atribuímos o URL de destino da informç ão a ser coletada. METHOD onde atribuímos o método de envio dos dados, pode ser GET em que os dados são enviados pela URL da página ou POST em que os dados são enviados por meio de uma variável oculta aos olhos dos usuários. E NAME que define o nome do formulário.

A entrada dos dados

A tag utilizada é a < INPUT> e dentro dela definimos:

  • TYPE que pode ser de vários tipos: text, hidden (que permite que um dado seja submetido sem que o usuário veja), password, file (o file só funcionará corretamente se o atributo ENCTYPE estiver com valor "multipart/form-data"), checkbox (caixa de multíplas opçãos), radio (caixa de opção única), submit (que é um botão para enviar os dados), reset, buttom (botão genérico normalmente usado para executar funções em linguagem client side como JavaScript) e image (como o buttom mas com uma imagem associada).

  • NAME atribui nome ao dado, é um campo obrigatório.

  • VALUE define o que deverá ser exibido no espaço, quando o dao Ã© tipo text e password não é obrigatório. Quando é hidden, checkbox e radio é obrigatório pode atribuir "ok". Quando é submit, reset e buttom atribuimos o texto que será exibido no botão.

  • SIZE define o tamanho da aba de entrada de dados. Pode ser utilizado nos tipos text, password e file.

  • MAXLENGTH determina o número máximo de caracteres que podem ser digitados na aba de entrada. Pode ser utilizado nos tipos text, pasword e file.

  • SRC, quando definimos o tipo IMAGE utilizamos o SRC para atribuirmos o nome da imagem a ser exibida. Nesse botão IMAGE, todos os demais atributos de < IMG> podem ser utilizados.

  • CHECKED, esse atributo é colocado junto a um INPUT com tipo checkbox ou radio, não possuí valor e determina que o campo esteja já pré selecionado.

  • READONLY, literalmente é apenas para ser visualizado pelo usuário, sem que este possa altera-lo. Podemos colocar uma informação em VALUE e colocar READONLY no fim do INPUT.
