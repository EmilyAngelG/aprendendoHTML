Portanto, podemos usar tantlo o < br> quanto o < p>, porém o < p> diferente do < br> é uma tag contêiner, ou seja, tem abertura e fechamento deixando o código mais organizado.

Diferente do < br> o < p> também pode usar o align para definir onde ficará a frase, aqui está em right.

Se você quer chegar onde a maioria não chega, faça o que a maioria não faz. O homem é aquilo que a sociedade faz dele. A educação não muda o mundo, a educaç ão muda pessoas, pessoas mudam o mundo. Ninguém pode voltar atrás e fazer um novo come ço mas todos podem mudar agora e fazer um novo fim.

LISTAS

Temos três tipos que listas: as definidas que não tem nenhum tipo de marcação e só podem ter duas camas, as listas não ordenadas que tem marcaço mas não tem ordem numérica, e temos as listas ordenadas, essas tem ordem numérica.

LISTA DE DEFINIÇÃO:

Aqui temos o contêiner < dl> e dentro dele temos o < dt> para apresentar o título, e dentro dele o < dd> para apresentar os subconteúdos. Exemplo:

Lista para o supermercado
Alimentos:
Arroz
Feijão
macarrão
Para limpeza:
Aguá sanitária
Sabão em pó

LISTA NÃO-ORDENADA

Nessa temos o contêiner < ul> e os marcadores < li> e podemos adicionar sublistas dentro da lista princi pal, utilizando mais < ul> e < li>. O < ul> é que inicia a lista mas os tópicos são escritos dentro dos < li>. Aqui temos marcadores, então para defini-los usamos TYPE dentro do < ul> geral ou dentro de cada < li>, os TYPES podem ser circle, square, e disc e caso não sejam definidoa o browser irá defini-los. Exemplo: essa lista está sem TYPE.

Lista para o supermercado

  • Alimentos:
    • Arroz
      • Arroz branco ou parbolizado
    • Feijão
      • preto ou carioca
    • macarrão
  • Para limpeza:
    • Aguá sanitária
    • Sabão em pó
  • Para sobremesa
    • Chocolate 70%
    • Leite moça

LISTA ORDENADA:

Essa é a mais organizada, pois com ela podemos elencar uma ordem para os itens da lista. Aqui o cont êiner é o < ol>, dentro dele temos < li> e podemos ter novos < ol> e < li> para sublistas, assim como na lista não ordenada. Aqui também podemos definir o TYPE, ele pode ser "1", "I", "i", "A" ou "a". Esse TYPE é definido no < ol>.

Lista de supermercado

  1. Alimentos: (TYPE automático sem definição)
    1. Arroz
    2. Feijão
    3. Macarrão
  2. Para limpeza: (TYPE="I")
    1. Água sanitária
    2. Sabão em pó
  3. Para sobremesa: (TYPE="a")
    1. Chocolate 70%
    2. Leite condensado
  4. Para os pets: (TYPE="i")
    1. Ração de cachorro
    2. Ração de gato
    3. Brinquedos

Podemos também alterar a ordem das numeraçãoes com dois outros comandos da lista ordenada. O START pode ser colocado junto com o < ol> e ele definirá o início da numeração. O VALUE defini um novo começo no meio da lista e pode ser colocado nos < li>. Exemplos:

  1. Para sobremesa: (TYPE="a" START="3" na principal e "4" na de dentro)
    1. Chocolate 70%
    2. Leite condensado
  2. Para os pets: (TYPE="i")
    1. Ração de cachorro
    2. Ração de gato
    3. Brinquedos(VALUE="9")
    4. Petiscos

LINHAS HORIZONTAIS

Vamos agora ver como são feitas as linhas aqui em HTML. A tag é < hr> e não é contêiner, nela teremos WIDTH que define a largura em pixels ou porcentagem da linha, SIZE que define a expessura em pixels, ALIGN que alinha em LEFT, RIGHT ou CENTER, COLOR que define a cor da linha. Exemplos:

width="100%" size="2" align="center" color="blue"

width="50%" size="10" align="left" color="pink"

LINKS E IMAGENS

Para trazer imagens ao HTML precisamos das seguintes estruturas. A tag < a> é o contêiner, nela temos HREF onde colocamos o link propriamente dito e TARGET que define onde esse link será aberto, pode ser _blank (em outra guia) ou _self (na mesma guia, substituindo-a). Exemplos:

< a> href="..." target="_blank">Link para a 1° parte< /a>

Link para a 1° parte

Imagens: A tag é < img src> a tag < img> não é contêiner e deve ser seguida obrigatóriamente da tag < src> pois é ela que determina qual imagem será utilizada.
Lembre-se que toda e qualquer referência, com exceção de links, deve ser carregada no projeto no ambiente de desenvolvimento, portanto a imagem a seguir se encontra na mesma pasta de projeto na IDE PyCharm onde este código está sendo escrito.

Para manipular nossa imagem utilizamos as tags: WIDTH para definir o tamanho em pixels ou em porcentagem da nossa imagem em largura, HEIGHT para definir em pixels ou em porcentagem a altura (não é uma boa idéia utilizar o WIDH e o HEIGHT ao mesmo tempo pois se um vai alterar a largura e outro a altura, a imagem pode ficar estranha), VSPACE para definir o espaç amento vertical da imagem e HSPACE para definir o espaçamento horizontal da imagem.

Além desses, temos que pensar também no texto, o ALIGN define de o texto ficará a LEFT ou a RIGHT da imagem.

Arthur Casas: “o modernismo envelheceu”

Ao mencionar a arquitetura contemporânea é impossível não citarmos o arquiteto brasileiro Arthur Casas, que fincou a sua visão peculiar do modernismo em nível nacional e internacional. O arquiteto dirige o Studio Arthur Casas com escritórios nas cidades de São Paulo e Nova York. Entre as inúmeras características que tornam Arthur Casas um arquiteto de referência está a valorização da liberdade criativa. O designer não suporta o universo clichê do neoclassicismo e a sensação de falso conforto revestida de materiais de péssima qualidade.

Avesso às reproduções do neoclassicismo, o arquiteto considera que o estilo tem pouca relação com a cultura brasileira, nosso clima e história. Para o arquiteto, por trás das construções que querem aparentar luxo e glamour no neoclássico, estão materiais de péssima qualidade, o que torna esse tipo de projeto bastante popular por conta do baixo custo, tornando-se um interessante investimento para as incorporadoras.

No lugar de apenas reproduzir tendências e imitar estilos ultrapassados, por que não adaptá-las a um novo cenário? Nesse contexto, o arquiteto considera o grande erro do modernismo contemporâneo deixar em segundo plano o conforto e a vida das pessoas. Concreto aparente, metal, vidro e formas cubistas fazem parte de um modernismo incompleto, segundo a visão de Arthur Casas.

É certo que a arquitetura de Casas possui forte influência dos projetos de Mies van der Rohe e Le Corbusier, incluindo a presença de pilotis, integração dos espaços, planos de vidro e projetos transparentes com a intenção de aconchegar todos os elementos envolvidos, todos conceitos dos cinco pontos da arquitetura moderna. Mas um dos requisitos do arquiteto em seus projetos é transformar o modernismo na arquitetura, que por excelência deve primar pelo conforto e responsabilidade ambiental.

É a união da natureza, do lar e das pessoas que o habitam. Veja mais detalhes sobre Mies van der Rohe, um dos arquitetos que inspiraram os projetos de Arthur Casas:
