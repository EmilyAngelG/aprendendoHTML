1. Aqui aprenderemos como fazer formulários na página web.

Para estruturar um formulário, usaremos a tag contêiner < form> e dentro dela iremos definir os parâmetros do nosso formulário, são eles

Vamos começar fazendo o formulário mais simples possível, veja esse exemplo: 

    < form>
        < p>Nome: < input type="text">< /p>
        < p>Sobrenome: < input type="text">< /p>
    < /form>

RESULTADO

Nome:

Sobrenome:

Observe que esse formulário recebe dados em formato de texto e no momento não está fazendo nada com esses dados que são digitados pelo usuário.

Agora observe este outro exemplo: 

    < form>
        < p>Nome: < input type="text" name="nome">< /p>
        < p>Sobrenome: < input type="text" name="sobrenome">< /p>
    < /form>

Nome:

Sobrenome:

Repare que agora, o navegador já está sugerindo palavras relacionadas ao que está sendo solicitado no formulário. Isso aconteceu porque dessa vez atribuímos um name ao input, esse name não tem a ver com o título da linha do formulário, esse é o nome que está atribuído em name. Como atribuímos o name "nome" o navegador buscou no seu histórico outros formulários que já preenchemos que tinham esse mesmo name e sugeriu as mesmas palavras que usamos antes. Se quiser é possível desativar essa função.

Desativando o preenchimento automático:

Para isso vamos adicionar a tag form um atributo chamado autocomplete e colocamos "off". 

        < form autocomplete="off">
            < p>Nome: < input type="text" name="nome">< /p>
            < p>Sobrenome: < input type="text" name="sobrenome">< /p>
        < /form>

Nome:

Sobrenome:

Enviando os dados

Certo, agora vamos aprender a enviar os dados recebidos no formulário, é bem simples. Basta criar um botão para enviar os dados, ou seja submete-los, da seguinte forma: 

        < form action="cadastro.js">
            < p>Nome: < input name="nome">< /p>
            < p>Sobrenome: < input name="sobrenome">< /p>
            < input type="submit" value="Enviar">
        < /form>

Nome:

Sobrenome:

Veja que o atributo action definido no form é ativado quando o usuário clica no botão de submissão e envia os dados a outro documento que irá fazer o tratamento deles, nesse caso com uma liguagem de programação. Definimos como exemplo, que os dados seriam enviados para um arquivo em JS chamado cadastro, contudo, como na verdade não criamos esse arquivo a página vai levar o usuário a um erro. Posteriormente veremos como realizar o tratamento desses dados ok.

Usando LABEL

Um detalhe muito importante na hora de fazer um formulário, é que precisamos relacionar o nome do espaço para digitação com o próprio espaço. Nos nossos exemplos anteriores não fizemos isso, ou seja, quando escrevemos "nome:" e então chamamos um input e o nome dele também era "nome" um não tinha a ver com o outro. Para fazermos essa relação é preciso utilizar o LABEL que é bem simples, veja como é: 

        < form action="cadastro.js">
            < p>
                < label for="inome">Nome: < /label>
                < input type="text" name="nome" id="inome">
            < /p>
            < p>
                < label for="isobrenome">Sobrenome: < /label>
                < input type="text" name="sobrenome" id="isobrenome">
            < /p>
            < p>
                < input type="submit" value="Enviar">
            < /p>
        < /form>

Aparentemente nada mudou, porém, se você clicar em cima da palavra "nome: " ou "sobrenome: " o espaço para digitação será acionado, pois, agora eles estão relacionados. Isso acorre porque vinculamos o nome que queriamos escrever ao id no espaço de digitação atravéd do LABEL. A utilidade disso é mais visivel em dispositivos móveis como celulares que tem um espaço menor para clicar, com essa associação o usuário pode digitar mesmo se clicar errado, no nome ao invés de na caixa de formulário. Importante dizer, utilizar o LABEL não é facultativo, é necessário sempre que estiver fazendo um formulário como esse.


2. E para onde vão os dados?

Bom, observe ainda mais um coisa importante. Quando o usuário clica em submeter os dados, repare que eles ficam em exibição na url na página. Isso não é extamente um problema, pois, isso ocorre mesmo como uma forma de direcionar a página. Porém, se o formulário solicitar senhas elas serão exibidas ali e ai será um problema. Vamos então cuidar disso.

Metodos GET e POST

Veja, o form tem method e ele pode ser de dois tipos GET ou POST, quando não definimos o method ele automáticamente utilizará o GET que é o que esteve sendo utilizado até agora nos nossos formulários de exemplo. Mas nós podemos colocar POST no method e veremos que os dados passarão a não serem mais exibidos na url. Veja o exemplo: 

        < form action="cadatro.js" method="post">
            < p>
                < label for="inome">Nome: < /label>
                < input type="text" name="nome" id="inome">
            < /p>
            < p>
                < label for="isobrenome">Sobrenome: < /label>
                < input type="text" name="sobrenome" id="isobrenome">
            < /p>
            < p>
                < input type="submit" value="Enviar">
            < /p>
        < /form>

Cabe resaltar, o POST é um método que não exibe os dados na url, porém, isso não quer dizer que esses dados estejam protegidos. O POST não está protegendo os dados, só não os está exibindo diretamente na url, mas, eles podem ser acessados de outras formas. Para proteger de fato dados, é necessário criptografá-los e isso é um outro aprendizado que não diz respeito ao que estamos aprendendo agora.

Quando usar o GET e o POST

É preciso usar o POST quando os dados solicitados são sensíveis e não queremos que sejam explícitamente exibidos na url, quando forem fotos e vídeos e quando o volume de dados últrapassar 3.000 bytes que é equivalente a aproximadamente 3.000 caracteres. Lembrando novamente que POST não é indicado para proteger dados importantes como números de cartões e senhas, para isso também é preciso aplicar criptografia.

4. Validação

Faça um teste novamente e perceba, mesmo não digitando nada nas caixas de formulário a submissão é aceita e a página de tratamento é chamada. Isso ocorre porque não estamos solicitando nenhum requerimento obrigatório, mas, podemos solicita-lo dentro do input da seguinte forma: 

        < form action="cadastro.js" method="post">
            < p>
                < label for="iusuario">Usuário: < /label>
                < input type="text" name="usuario" id="iusuario" required>
            < /p>
            < p>
                < label for="isenha">Senha: < /label>
                < input type="password" name="senha" id="isenha" required>
            < /p>
            < p>
                < input type="submit" value="Enviar">
            < /p>
        < /form>

Essas caixinhas que aparecem solicitando os dados são pequenos trechos de JS executados pelo próprio navegador a mando do required.
